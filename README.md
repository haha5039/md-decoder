# 마스터듀얼 카드 디코더 (Master Duel Card Decoder)

> **💡 바로 플레이하기 (GitHub Pages)**: 👉 **[https://haha5039.github.io/md-decoder/](https://haha5039.github.io/md-decoder/)**

유희왕 마스터듀얼(Yu-Gi-Oh! Master Duel)의 미니게임 이벤트인 **'카드 디코더(Card Decoder)'**를 완벽하게 보조하기 위해 제작된 웹 기반 지능형 분석 도구입니다. 사용자가 게임 내에서 얻은 판정 결과(O/X)를 입력하면, 13,000장 이상의 전체 유희왕 카드 데이터베이스를 실시간으로 스캔하여 정답일 확률이 가장 높은 카드를 수학적으로 계산해 추천해 줍니다.

---

## 📖 미니게임 "카드 디코더(Card Decoder)" 상세 룰

'카드 디코더'는 정답이 되는 **'비밀 카드 1장'**의 정체를 제한된 기회 안에 유추해내는 게임입니다. 
플레이어가 임의의 카드 1장을 추측하여 제출하면, 게임 시스템은 제출한 카드와 비밀 카드 간의 **6가지 고유 스탯**이 일치하는지 여부를 각각 `O(일치)` 또는 `X(불일치)`로 피드백해 줍니다.

### 🔍 6가지 검증 스탯 (Stats)
1. **카드 프레임 (Frame Type)**: 일반, 효과, 융합, 싱크로, 엑시즈, 링크, 펜듈럼, 마법, 함정 등 카드의 기본 테두리 색상.
2. **속성 (Attribute)**: 빛, 어둠, 땅, 물, 화염, 바람, 신, 그리고 마법/함정 고유 속성.
3. **레벨 / 랭크 / 링크 (Level/Rank/Link)**: 몬스터의 별 개수 혹은 링크 마커 수.
4. **종족 (Race / Type)**: 드래곤족, 악마족 등 몬스터 종족이나 마법/함정 카드의 상세 분류(지속, 속공 등).
5. **공격력 (ATK)**: 몬스터의 원래 공격력 수치 (마법/함정은 해당 없음).
6. **수비력 (DEF)**: 몬스터의 원래 수비력 수치 (링크 및 마법/함정은 해당 없음).

### ⚠️ 마스터듀얼 특수 룰 완벽 호환 (Special Rules)
본 어플리케이션은 단순한 텍스트 매칭을 넘어 마스터듀얼 시스템상의 예외 룰을 완벽하게 계산합니다.
* **펜듈럼 프레임 호환**: '펜듈럼/싱크로' 카드는 '펜듈럼' 단서와 '싱크로' 단서 양쪽 모두에 대해 `O(일치)`로 판정되는 하이브리드 로직을 적용합니다.
* **하이브리드 레벨 판정**: 룰 상 레벨/랭크가 카드 이미지의 숫자와 다르게 취급되는 카드들(`얼티마야 촐킨`, `FNo.0 미래황 호프` 등)은, 실제 이미지 레벨과 룰상 레벨 중 어느 하나만 일치해도 `O`로 취급되는 공식 룰을 배열(Array) 매칭 알고리즘을 통해 구현했습니다.

## 🧠 핵심 기술 및 알고리즘 상세 (Technical Details)

본 프로젝트는 조건에 맞는 카드를 단순히 '나열'하는 것에 그치지 않습니다. 남은 후보 카드 전체를 대상으로, **어떤 카드로 도전해야 다음 턴에 가장 확실하게 정답을 좁힐 수 있는지** 4가지 수학적 지표를 실시간으로 연산하여 최적의 스나이핑 및 정찰(Scout) 픽을 제시합니다.

### 1. 4가지 지능형 추천 로직 (Multi-Judgment Logics)
각 카드별로 피드백 가능한 64가지(2^6)의 O/X 조합 분기(Buckets)를 모두 시뮬레이션하여 다음 4가지 지표를 산출합니다.

* **기대 정보량 (Shannon Entropy)**
  * **공식**: $H = -\sum_{i=1}^{64} P(b_i) \log_2 P(b_i)$
  * **기술적 원리**: 정보 이론(Information Theory)에 기반하여 후보 풀을 여러 버킷으로 가장 고르게 분산시키는 카드를 찾습니다. 엔트로피(Bits)가 가장 높은 카드를 선택하면 평균적으로 최단 횟수 내에 정답에 도달할 수 있습니다.
* **최악 상황 최소화 (Minimax)**
  * **공식**: $\min(\max(BucketSize))$
  * **기술적 원리**: 시뮬레이션 된 64개 분기 중 가장 많은 카드가 몰려있는 분기(최악의 상황)의 크기가 가장 작은 카드를 찾습니다. 단 한 번의 불운한 판정으로 수백 장의 후보가 남는 대참사를 알고리즘 단위에서 원천 봉쇄합니다.
* **단판 확정 확률 (One-shot Isolation)**
  * **공식**: $(Count\_of\_Buckets\_with\_Size\_1) / Total\_Candidates$
  * **기술적 원리**: 다음 힌트 결과로 남는 후보가 정확히 '1장'이 되거나, 제출한 카드 자체가 정답일 확률의 합산입니다. 도전 횟수가 1회밖에 남지 않았을 때 시스템이 동적 가이드라인을 통해 이 로직을 강력히 권장합니다.
* **평균 잔여 최소화 (Expected Remaining)**
  * **공식**: $\sum (BucketSize_i^2) / N$
  * **기술적 원리**: 다음 도전을 마친 직후 최종적으로 남게 될 후보 카드 개수의 수학적 기댓값을 직접적으로 최소화하는 가장 직관적이고 안정적인 지표입니다.

### 2. 브라우저 사이드 대규모 연산 최적화 (Optimization)
자바스크립트 엔진이 브라우저의 메인 스레드에서 13,000+장의 카드를 프리징(멈춤 현상) 없이 연산하기 위해 고도화된 최적화 기법이 적용되었습니다.
* **통계적 샘플링(Statistical Sampling)**: 후보군이 800장을 넘어갈 경우, 1천만 번 이상의 반복 연산이 발생합니다. 이를 방지하기 위해 전체 후보군에서 800장의 대표 샘플을 균등 추출(Uniform Sampling)하여 평가 풀을 축소함으로써, 통계적 정확도를 유지하면서도 O(1) 수준의 렌더링 속도(50ms 미만)를 달성했습니다.
* **사전 캐싱(Pre-caching)**: 레벨 파싱 배열(`validLevels`) 등 반복되는 함수 호출 결과를 앱 로딩(Init) 단계에서 전체 카드 객체에 메모이제이션(Memoization)하여 O(N^2) 루프의 병목을 완벽히 제거했습니다.

## 📊 데이터 출처 및 API (Data Source)

본 프로젝트의 엔진 구동을 위한 기초 카드 데이터는 전 세계적으로 가장 공신력 있는 **[YGOPRODeck API](https://db.ygoprodeck.com/api-guide/)**에서 추출되었습니다.
* **데이터 가공 과정**: `https://db.ygoprodeck.com/api/v7/cardinfo.php` 에서 받아온 약 30MB의 거대한 원본 JSON 데이터에서, 카드 디코더 연산에 불필요한 플레이버 텍스트, TCG/OCG 금제 리스트, 마켓 가격 정보 등을 모두 쳐내었습니다.
* **최종 데이터셋**: 오직 이름, 이미지 링크(`image_url`), 그리고 판정에 필요한 6가지 핵심 스탯만을 남긴 초경량 배열(`cards_data.js`)로 압축하여 클라이언트의 로딩 시간과 메모리 점유율을 획기적으로 낮췄습니다.

## 💻 로컬 개발 및 배포 환경 (Development & Deployment)

본 웹 어플리케이션은 최신 프론트엔드 빌드 툴체인을 통해 개발 및 자동 배포됩니다.

### 기술 스택 (Tech Stack)
- **UI & Logic**: HTML5, Vanilla JavaScript (ES6+), Vanilla CSS3
- **Dev Server & Bundler**: Vite (빠른 HMR 및 초경량 프로덕션 빌드 제공)
- **CI/CD**: GitHub Actions (자동화된 파이프라인으로 `main` 브랜치 Push 시 자동 빌드)
- **Hosting**: GitHub Pages

### 로컬 실행 방법 (Setup)
로컬 환경에서 직접 소스 코드를 실행하고 테스트하려면 아래 단계를 따르세요. (Node.js v18 이상 권장)

1. **저장소 클론 및 폴더 이동**
   ```bash
   git clone https://github.com/haha5039/md-decoder.git
   cd md-decoder
   ```
2. **패키지 설치**
   ```bash
   npm install
   ```
3. **개발 서버 가동**
   ```bash
   npm run dev
   ```
   > 터미널에 출력되는 `http://localhost:5173/` 링크를 클릭하여 접속할 수 있습니다.
4. **프로덕션 빌드**
   ```bash
   npm run build
   ```
   > 배포에 최적화된 파일들이 `dist/` 폴더 내에 생성됩니다.
