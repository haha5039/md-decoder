# 마스터듀얼 카드 디코더 (Master Duel Card Decoder)

유희왕 마스터듀얼(Yu-Gi-Oh! Master Duel)에서 주어지는 힌트(카드 프레임, 속성, 레벨/랭크/링크, 종족, 공격력, 수비력)를 기반으로 정답 카드를 빠르고 정확하게 찾아낼 수 있도록 돕는 웹 기반 분석 도구입니다.

## 🚀 실시간 플레이
👉 **[웹앱 바로가기](https://haha5039.github.io/md-decoder/)**

## 📖 이벤트 룰 (Card Decoder Event Rules)
이 도구는 마스터듀얼에서 진행되는 **카드 디코더(Card Decoder)** 미니게임 이벤트의 규칙을 따릅니다.
- **게임 방식**: 정답이 되는 '비밀 카드'를 맞히기 위해 플레이어가 임의의 카드를 제출(도전)합니다.
- **결과 판정**: 제출한 카드와 비밀 카드의 6가지 스탯(**카드 프레임, 속성, 레벨/랭크/링크, 종족, 공격력, 수비력**)이 각각 일치하는지 여부(O/X)를 피드백으로 받습니다.
- **목표**: 제한된 기회 안에 얻은 피드백들을 단서로 삼아 정답 카드를 정확하게 유추해내는 것이 목표입니다.

## 📊 카드 데이터 출처 (Data Source)
본 프로젝트에 사용된 13,000+장의 모든 유희왕 카드 데이터베이스 및 카드 이미지는 **[YGOPRODeck API](https://ygoprodeck.com/api-guide/)**를 기반으로 수집 및 가공되었습니다. 

## ✨ 주요 기능

이 프로젝트는 단순한 조건 필터링을 넘어, 남은 도전 횟수와 카드 풀에 맞추어 **가장 최적화된 스나이핑 및 정찰 카드**를 추천하는 고도화된 수학적 알고리즘을 제공합니다.

### 4가지 지능형 추천 알고리즘 (Judgment Logics)
상황에 맞추어 4가지의 알고리즘을 탭 클릭으로 실시간 전환하며 활용할 수 있습니다.
1. **기대 정보량 극대화 (Shannon Entropy)**: 카드를 가장 고르게 여러 그룹으로 쪼개어 가장 균등하게 후보를 줄이는 '스탠다드 추천'입니다.
2. **최악 상황 최소화 (Minimax)**: 어떤 불운한 판정이 나오더라도 남을 최대 카드 개수를 최소화하여 극단적인 리스크를 차단합니다.
3. **단판 종결 확률 (One-shot Isolation)**: 다음 1번의 힌트만으로 후보가 정확히 1장이 되거나 즉시 정답을 맞출 수 있는 도박수/스나이핑 전용 추천입니다. (기회가 얼마 남지 않았을 때 유용)
4. **평균 잔여 최소화 (Average)**: 남게 될 카드의 수학적 기댓값(평균)을 가장 작게 줄여주는 가장 직관적인 추천입니다.

### 동적 전략 가이드라인 제공
입력된 '남은 도전 횟수'와 '남은 힌트 수', 그리고 남은 후보의 숫자를 실시간으로 분석하여 다음과 같이 조언합니다.
- *"도전 기회가 부족하므로 힌트를 개방하세요."*
- *"기회가 넉넉하니 기대 정보량이 높은 극한의 정찰 픽을 던져보세요."*
- *"기회가 없으니 단판 확정 확률 픽으로 도박을 해야 합니다."*

### 마스터듀얼 특수 룰 완벽 적용
- **하이브리드 레벨 판정**: `패왕룡 즈아크`, `얼티마야 촐킨`, `FNo.0 미래황 호프` 등 룰상 취급되는 레벨/랭크와 실제 이미지의 레벨이 다른 특수 카드들에 대해 정확한 듀얼 판정을 지원합니다.
- **카드 프레임 호환**: 펜듈럼 몬스터와 일반/효과/엑스트라 덱 몬스터 간의 복합 프레임 판정을 완벽히 반영합니다.

## 💻 로컬 환경 실행 방법 (Development Setup)

이 프로젝트는 **Vite**를 기반으로 구축되었습니다.

### 사전 요구 사항
- Node.js (v18 이상 권장)
- npm

### 설치 및 실행
1. 저장소 클론
   ```bash
   git clone https://github.com/haha5039/md-decoder.git
   cd md-decoder
   ```
2. 의존성 설치
   ```bash
   npm install
   ```
3. 로컬 개발 서버 실행
   ```bash
   npm run dev
   ```

## 🛠️ 기술 스택 (Tech Stack)
- **Frontend Core**: HTML5, Vanilla JavaScript (ES6+), CSS3
- **Build Tool**: Vite
- **Deployment**: GitHub Actions & GitHub Pages
- **Data Optimization**: 13,000+장의 카드 데이터를 클라이언트 측에서 병목 없이 처리하기 위한 통계적 샘플링(Sampling) 및 사전 캐싱(Pre-caching) 최적화 기법 적용.
